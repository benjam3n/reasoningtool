# ARAW 16x: Universal Principles of Writing Quality

## Key Findings Summary

### 1. Quality is Relational, Not Intrinsic

**FOUNDATIONAL FINDING**: Writing quality is NOT a property of the text itself. Quality is a four-place relation:

```
Quality = f(text, reader, context, purpose)
```

The current skill treats quality as f(text). This is a category error.

**Implication**: Before defining criteria, must specify reader, context, and purpose.

---

### 2. Operational Floor vs Non-Operational Ceiling

**FOUNDATIONAL FINDING**: Quality has two levels:

| Level | Type | Examples | Skill Can Handle? |
|-------|------|----------|-------------------|
| FLOOR | Necessary conditions | Spelling, grammar, length | YES |
| CEILING | Sufficient conditions | Elegance, insight, impact | NO (requires judgment) |

The skill can guarantee floor, not ceiling. It should not overpromise.

---

### 3. "MUST" is Ambiguous

**FOUNDATIONAL FINDING**: "MUST" conflates two concepts:

- **Necessary**: Without it, output fails
- **Jointly sufficient**: Together with other MUSTs, guarantees success

**Definition**: MUST = "necessary condition such that the set of all MUSTs is jointly sufficient for success"

**Alternative question**: "What would cause this writing to fail?" (pre-mortem framing)

---

### 4. Success Has Multiple Definitions

Five candidate definitions of success:

| Definition | Success Means | Measurable? |
|------------|---------------|-------------|
| Purpose fulfilled | Writing does what it was meant to do | Requires knowing purpose |
| Reader state change | Reader's state changes appropriately | Hard without reader |
| Criteria satisfaction | All criteria are met | Yes, but may miss actual success |
| Counterfactual improvement | Better than not writing | Requires comparison |
| Communication fidelity | Reader receives what writer sent | Like signal-to-noise |

**Implication**: Skill should require explicit success definition.

---

### 5. Six Universal Principles

| Principle | Description | Form |
|-----------|-------------|------|
| Information-to-effort ratio | Quality ∝ Information / Effort to extract | Ratio |
| Expectation match | Quality ∝ Match between expectation and delivery | Match |
| Cognitive load minimization | Quality ∝ 1 / Cognitive load | Ratio |
| Signal-to-noise ratio | Quality ∝ Signal / Noise | Ratio |
| Fit-for-purpose | Quality = Fitness for purpose | Match |
| Minimal distortion | Quality ∝ 1 / Distortion | Ratio |

**Common structure**: All are ratios or matches. Quality is always comparative.

---

### 6. Clarity Decomposes into Six Types

| Type | Definition | Test |
|------|------------|------|
| Structural | Organization is evident | Reader can outline the piece |
| Linguistic | Sentences parse easily | No re-reading required |
| Semantic | Meaning is unambiguous | Multiple readers extract same meaning |
| Conceptual | Ideas are accessible | Reader can teach the concept |
| Pragmatic | Purpose/action is obvious | Reader can state the point |
| Referential | References are clear | Reader can identify all referents |

**Implication**: Don't say "be clear" - specify which type.

---

### 7. Quality Primitives (Explicit)

Seven primitives the skill should specify:

1. **Information density**: Information per word
2. **Processing fluency**: Ease of understanding
3. **Precision**: Specificity of claims
4. **Structure**: Logical organization
5. **Relevance**: Connection to reader's goals
6. **Actionability**: Can reader act on it
7. **Defensibility**: Can claims be supported

---

### 8. Ten Quality Guesses

| # | Guess | Formalization |
|---|-------|---------------|
| 1 | Efficiency of meaning transfer | Q = Meaning / Effort |
| 2 | Appropriate reader state change | Q = similarity(actual, intended) |
| 3 | Expectation satisfaction | Q = met / total expectations |
| 4 | Defect absence | Q = 1 - (defects / possible) |
| 5 | Purpose fit | Q = achieved / intended purpose |
| 6 | Cognitive fluency | Q ∝ 1 / cognitive_load |
| 7 | Information density | Q = information / words |
| 8 | Trust generation | Q = trust generated |
| 9 | Memorability | Q ∝ retention over time |
| 10 | Re-read value | Q ∝ value from re-reading |

---

### 9. Question Sequence (Better than Single Question)

Replace "What MUST be true?" with:

1. Who is the reader?
2. What should change for them?
3. What would prevent that?
4. What must be true to prevent failure?
5. How would we verify?

- **Resolution**: Context-dependent selection

- **Resolution**: Explicit primitives, flexible combination

## Meta-Analysis

This 16x ARAW discovered that the writing skill has foundational problems:

1. It treats quality as intrinsic when it's relational
2. It conflates floor (necessary) with ceiling (sufficient)
3. Its core question ("What MUST be true?") is ambiguous
4. It lazily delegates quality definition to the LLM

The fix is not "add more phases" but "reconceptualize quality as relational" and "provide explicit primitives and guesses."

The universal principles discovered (all ratios/matches) suggest that quality is always comparative, never absolute. This is philosophically significant: there is no "good writing" in isolation, only "good writing for this reader, context, and purpose."
